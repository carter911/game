Introduction

This document is intended to assist software developers who want to develop applications using UTDeliver.com API. It fully documents our application programming interface (API) that allows developers to interact with our system.

To use the information provided here you should be familiar with:

  • RESTful web services
  • HTTP/1.1
  • JSON (JavaScript Object Notation)

Our Application Program Interface will allow you to automate the process of searching/buying player auctions and comfort trades within your own software.

Since our API is based on REST principles, it's very easy to write and test applications. You can use your browser to access URLs, and you can use pretty much any HTTP client in any programming language to interact with the API.

All responses from our API are in JSON format.

Requests are limited at 60 RPM (Requests per minute).

Authentication

In order to use our API, you must have an active account at UTDeliver.com. The authentication is based on "HTTP Basic auth", making it dead simple to make calls on our platform. The following is an example with PHP, just one line of code is required for authentication.

curl_setopt($ch, CURLOPT_USERPWD, '{username}:{password}');

Our REST API is served over HTTPS. To ensure data privacy, unencrypted HTTP is not supported. The Base URL for all the requests is:

https://www.utdeliver.com/api/20

Coin Balance

This section describes how to fetch your account coin balances.

Resource URI

https://www.utdeliver.com/api/20/balance

Method

GET

Response

A successful call will return a JSON object.

{
  "ps4": 1000000,
  "ps3": 50000,
  "xboxone": 1200000,
  "xbox360": 10000
}
Name Type Description
<platform> Integer Coin balance for <platform>.

Coin Prices

This section describes how to get the current coin prices.

Resource URI

https://www.utdeliver.com/api/20/price/<platform>

Valid <platform> parameters are:

  • ps4
  • xboxone
  • ps3
  • xbox360

Method

GET

Response

A successful call will return a JSON object.

{
    "availability": "high",
    "price" : 0.0745
}
Name Type Description
availability String Human readable stock availability.
price Float Returns the price for 1k coins.

Buy Coins

This section describes how to buy coins.

Resource URI

https://www.utdeliver.com/api/20/buyCoins

Method

POST

POST Data

Name Type Description
platform String One of the following:

  • ps4
  • xboxone
  • ps3
  • xbox360
amount Integer Amount of coins to purchase. Min: 10000. Must be multiple of 10,000.
price Float Price of coins, as returned by previous call Coin Prices.

Response

A successful call will return a JSON object.

{
    "success" : "Bought 100,000 ps4 @0.075",
    "error" : "Coin prices has changed: 0.0745"
}

Add Comfort Trade

This section describes how to add a Comfort Trade.

Resource URI

https://www.utdeliver.com/api/20/comfort/add

Method

POST

POST Data

Name Type Description
platform String One of the following:

  • ps4
  • xboxone
  • ps3
  • xbox360
email String The Ultimate Team account email.
password String The Ultimate Team account password.
backupCode String Origin account backup code for Two Factor authentication.
amount Integer Amount of coins you want to fill the accounts with. Example (100k): 100000.
orderId String Optional: An identifier for this order.
ownStock String Optional: If you want to deliver this order using your own stock, you can specify this parameter to y.

Response

A successful call will return a JSON object. Example:

{
    "success" : true
}

A failed call will return two additional fields error containing a verbose explaination of the failure and an errorCode. This is a list of possible errorCode:

  • UNEXPECTED_EXCEPTION
  • ACCOUNT_ALREADY_ADDED
  • WRONG_EMAIL_OR_PASSWORD
  • WRONG_BACKUP_CODE
  • WRONG_SECURITY_ANSWER
  • TRANSFER_MARKET_DISABLED
  • INSUFFICIENT_COINS
  • TRADEPILE_FULL
  • INVALID_TWO_FACTOR_AUTH_MODE
  • WRONG_EMAIL_PASSWORD
  • EMAIL_PROVIDER_NOT_SUPPORTED
  • LIMIT_EXCEEDED

You must replace <email|couponKey> with the Ultimate Team email account or the coupon key you want to check the status of.

GET

Pause/Unpause/Remove Comfort Trade

This section describes how to pause / unpause / remove a Comfort Trade.

Resource URI

https://www.utdeliver.com/api/20/comfort/<email|couponKey>/status

Method

POST

POST Data

Name Type Description
action String One of the following:

  • pause to pause a Comfort Trade
  • remove to remove a Comfort Trade (you must have paused / completed a Comfort Trade before removing it)
  • run to restart a paused Comfort Trade

Response

A successful call will return a JSON object. Example:

{
    "success" : true
}

Update status of orders using your own stock

When you submit a new order and you have chosen to use your own stock it's your responsability to update the status on our system accordingly. Otherwise orders won't process correctly and your customers will be able to re-use existing coupon codes.

Resource URI

https://www.utdeliver.com/api/20/ownStock/submit

Method

POST

POST Data

Name Type Description
platform String One of the following:

  • ps4
  • xboxone
  • ps3
  • xbox360
tradeId Integer The Trade ID you are updating the status of.
action String One of the following:

  • success if you have already bought the provided Trade ID
  • fail if you could not find the player on the transfer market or any error occurred (the customer's card will be ignored by the system and a new one will be used)

Response

A successful call will return a JSON object. Example:

{
    "success" : true
}
